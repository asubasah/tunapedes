# Daily Log: 2026-04-13

## Aktivitas Resolusi FitMotor CRM (Sesi Multi-Tenant & Dashboard Frontend)
- **Database Alignment:** Memperbaiki bug salah sambung database (`fitmotor` ke `fitmotor_crm`) dan penyesuaian kolom `kode_booking` menjadi `id` yang membuat backend API Node.js sering hang dan menolak pembaruan data Kanban.
- **Isolasi Multi-Tenant Gagal via ENV, Sukses via URL Port:** Awalnya mencoba melempar variabel `VITE_CABANG` lewat BAT file secara dinamis, namun digagasi bahwa metode paling mutlak/bebas eror ("bulletproof") adalah secara `hardcode` mapping dari nomor port (5000=master, 5001=adiwerna, dst) dengan membaca `window.location.port` di `main.ts`. Hal ini secara permanen menuntaskan masalah "Akses Ditolak" pada dasbor cabang.
- **Patch N8N Flow `01` dan `05`:** Alur Cek Booking via WhatsApp sempat macet dan memunculkan error `device_id is required`. Diperbaiki dengan menginjeksi ekstraksi `device_id` dan `cabang_id` ke payload output dari `01_main_router.json` agar `05_cek_booking` sadar dari instance mana wa gatewaynya perlu membalas. 
- **Kanban Board Fix:** Meresolusi isu "Kartu Terbang/Hilang" saat Kanban digeser dari Pending ke Proses ke Selesai. Masalah utamanya adalah kolom "SELESAI" sebelumnya ditiadakan pada backend `(WHERE b.status != 'selesai')` dan tidak ada array handler-nya di Frontend. Sekarang Kanban sepenuhnya operasional dengan urutan _Newest on Top_ (`ORDER BY created_at DESC`).
- **Pembersihan Zombie Node:** Di-integrasikan `taskkill` langsung ke dalam `START_ECOSYSTEM.bat` agar restart stack selalu bersih dari memory leak / backend menggantung.
- **Push Notifikasi Otomatis (GoWA API):** Menginjeksi trigger WA notifikasi langsung pada backend `server.js` (endpoint `PUT /api/bookings/:id/status`). Tanpa perlu workflow N8N, pembaruan status Kanban dari Dashboard kini akan langsung menembak API GoWA sesuai `device_id` cabang dan mengirim notifikasi WhatsApp *real-time* ke pelanggan saat bergeser ke **Tandai Proses** atau **Selesai**.

**Status Saat Ini:**
Sistem sudah 100% stabil untuk produksi harian. Kanban, Dashboard, dan GoWA AI saling berkomunikasi dengan sempurna.
